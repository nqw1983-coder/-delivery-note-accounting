# 手写送货单网页控制面板实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 按已确认静态图实现网页版控制面板，并逐步补齐月份表、上传扫描模拟、入账确认和统计功能。

**Architecture:** 第一阶段做纯前端本地网页，不接真实后端，不保存原图，不调用真实识别服务。页面以一个控制面板为核心：左侧是年份、搜索、5 到 1 月有效月份、12 行月份区占位、底部上传扫描入口；右侧是当前月份完整 30/31 天金额表格。

**Tech Stack:** React + Vite + TypeScript + CSS Modules 或普通 CSS。图标优先使用 lucide-react。数据先放在前端内存里，后续需要持久化时再接本地存储或后端。

---

## 1. 当前已确认设计

本计划按以下文件和静态图反馈执行：

- 规格文档：`docs/superpowers/specs/2026-05-29-delivery-dashboard-architecture-design.md`
- 左侧月份列表：放在搜索框下面，显示 2026 年 5 月、4 月、3 月、2 月、1 月。
- 月份区域：保留 12 行高度，其余 7 行留空。
- 月份行：只显示月份和金额，不显示待补状态。
- 上传区：固定在左侧底部，和页面下端对齐，包含上传送货单图片、开始扫描、待补 2。
- 主表：显示整月记录，5 月显示 1 日到 31 日。
- 表头：`日期`、`吾湘`、`吾黄`、`醉得意`、`萍姐`、`柳`、两个空白列、`当日合计`。
- 两个空白列：表头和表格内容保持空白。
- 当日合计：永远是最后一列。
- 第一版不做：原图保存、查看原图、付款、价格单、导出、打印、语音、五页面菜单。

## 2. 我的建议

建议不要一上来把识别、入账、重复判断全做完。先把页面做成稳定的本地网页，再逐步加功能。

原因很简单：这个项目最重要的是老板看得懂、点得顺。如果先把功能堆进去，后面发现布局不顺，会返工很多。

推荐顺序：

1. 先实现静态控制面板。
2. 本地运行，用浏览器看 Mac 宽度和手机宽度。
3. 再做月份切换、加月份、月表计算。
4. 再做上传扫描的模拟流程。
5. 最后做确认入账、新店面、疑似重复这些规则。

## 3. 文件结构规划

建议新建一个前端项目，文件职责如下：

```text
package.json
vite.config.ts
index.html
src/
  main.tsx
  App.tsx
  styles.css
  data/
    seedData.ts
  types/
    dashboard.ts
  components/
    Sidebar.tsx
    MonthList.tsx
    UploadScanPanel.tsx
    MonthTable.tsx
    AnnualStatsDialog.tsx
    RecognitionDialog.tsx
    ConfirmDialog.tsx
```

各文件作用：

| 文件 | 作用 |
|---|---|
| `App.tsx` | 管理当前年份、当前月份、单据数据和主要弹窗 |
| `styles.css` | 全局布局、颜色、表格、按钮、手机适配 |
| `seedData.ts` | 2026 年 1 月到 12 月、默认店面、示例金额 |
| `dashboard.ts` | 年份、月份、店面、单据的数据格式 |
| `Sidebar.tsx` | 左侧整体结构 |
| `MonthList.tsx` | 5 到 1 月有效月份列表，并保留 12 行高度 |
| `UploadScanPanel.tsx` | 上传、开始扫描、待补提示 |
| `MonthTable.tsx` | 30/31 天金额表格 |
| `AnnualStatsDialog.tsx` | 年度统计弹窗 |
| `RecognitionDialog.tsx` | 识别结果确认弹窗 |
| `ConfirmDialog.tsx` | 新店面、疑似重复、取消结果等确认 |

## 4. 阶段一：控制面板静态布局

目标：先做一个像静态图的网页，功能可以暂时不完整，但布局必须对。

- [ ] 创建 React + Vite 前端项目。
- [ ] 安装基础依赖：`react`、`vite`、`typescript`、`lucide-react`。
- [ ] 创建左侧栏：标题、年度统计、加号、年份选择、搜索框。
- [ ] 创建月份列表：显示 2026 年 5 月到 1 月，2026 年 5 月选中。
- [ ] 月份列表区域保留 12 行高度，其余 7 行为空白占位。
- [ ] 月份行只显示月份和金额，不显示待补状态。
- [ ] 把上传送货单图片、开始扫描、待补 2 固定到左侧底部，并和页面下端对齐。
- [ ] 创建右侧主区域标题：2026年5月。
- [ ] 创建主表，显示 1 日到 31 日。
- [ ] 表格加两列空白列，放在 `柳` 和 `当日合计` 之间。
- [ ] 设置视觉风格：白底、浅灰线、深绿色主按钮、选中格绿色边框。

验收：

- 左侧没有五页面菜单。
- 上传扫描入口在左侧底部。
- 月份列表在搜索框下面。
- 月份区域有 12 行高度，其中 5 到 1 月有效显示，其余 7 行空白。
- 月份行没有待补文字。
- 主表能看到 31 天。
- 当日合计是最后一列。

## 5. 阶段二：本地运行和视觉验证

目标：网页能在本地打开，并确认桌面和手机宽度不乱。

- [ ] 启动本地开发服务器。
- [ ] 在浏览器打开本地地址。
- [ ] 截图桌面宽度。
- [ ] 截图手机宽度。
- [ ] 检查文字是否重叠。
- [ ] 检查表格是否能横向适配。
- [ ] 检查按钮是否容易点击。
- [ ] 根据截图修正间距、字体、列宽。

验收：

- Mac 浏览器能打开。
- 手机宽度不乱。
- 左侧栏和表格不互相挤压。
- 左侧底部上传扫描区贴近页面下端。
- 31 天记录可查看。
- 上传、扫描、待补提示清楚。

## 6. 阶段三：月份和表格数据功能

目标：让月份切换、加月份、金额合计真正生效。

- [ ] 点击月份后切换右侧标题和表格。
- [ ] 2026 年初始显示 5 月到 1 月。
- [ ] 月份区域保留 12 行高度，其余 7 行空白。
- [ ] 点击加号时，按规格新增下一个月份。
- [ ] 如果年份已有 12 月，再点加号提示当前年份已满 12 个月。
- [ ] 每个月自动带出默认店面：吾湘、吾黄、醉得意、萍姐、柳。
- [ ] 新店面确认后，以后新增月份默认带出。
- [ ] 计算当日合计。
- [ ] 计算本月合计。
- [ ] 空白列保持空白，不参与计算。

验收：

- 切换月份后表格变化。
- 加号规则正确。
- 当日合计正确。
- 本月合计正确。
- 空白列没有 0。

## 7. 阶段四：上传扫描模拟流程

目标：先做模拟识别，不接真实图片识别服务。

- [ ] 没选图片时，点击开始扫描提示先上传图片。
- [ ] 选择图片后显示文件名即可，不保存原图。
- [ ] 点击开始扫描后，生成一条模拟识别结果。
- [ ] 识别结果包含店面、日期、金额、备注。
- [ ] 识别不清楚字段留空。
- [ ] 不清楚时提示用户补充。
- [ ] 待处理只更新数量或简单提示，不做列表。

验收：

- 不会自动保存原图。
- 不会瞎填不清楚字段。
- 用户能看见哪些字段需要补。
- 待补只是简单提示。

## 8. 阶段五：确认入账和修改

目标：用户能把识别结果确认进月表，并能修改金额。

- [ ] 显示识别结果确认弹窗。
- [ ] 店面可选择已有店面。
- [ ] 日期可修改。
- [ ] 金额可修改。
- [ ] 备注可填写。
- [ ] 点击确认入账后写入对应日期、店面格子。
- [ ] 同一天同店面多张单，格子显示总金额。
- [ ] 点击金额格子，显示简单加法明细。
- [ ] 修改金额后只保留最新金额，页面不显示最后修改时间。

验收：

- 入账后月表数字更新。
- 多张单能累加。
- 格子明细能显示加法来源。
- 修改金额后合计更新。

## 9. 阶段六：确认规则

目标：把需要提醒确认的地方补齐。

- [ ] 新店面入账前弹窗确认。
- [ ] 新店面确认后加入当前月份店面列。
- [ ] 新店面以后新增月份默认带出。
- [ ] 同一天、同店面、同金额时提示可能重复。
- [ ] 疑似重复必须确认后才加入。
- [ ] 识别日期对应月份不存在时，不自动创建，不写入，提示先用加号建月份。
- [ ] 取消本次未入账识别结果前提示确认。

验收：

- 新店面不会静默加入。
- 重复单不会静默加入。
- 目标月份不存在时不会自动创建。
- 取消结果前有确认。

## 10. 阶段七：年度统计

目标：年度统计弹窗能看当前年份每个月总金额。

- [ ] 点击年度统计打开弹窗。
- [ ] 显示 1 月到 12 月。
- [ ] 每个月显示总金额。
- [ ] 点击某个月可切换到该月份。
- [ ] 待补数量只显示简单数字，不做展开列表。

验收：

- 年度统计金额来自月表。
- 点击月份能跳转。
- 弹窗清楚，不复杂。

## 11. 阶段八：最终验证

目标：确认页面真的能用，而不是只看代码。

- [ ] 桌面浏览器打开。
- [ ] 手机宽度检查。
- [ ] 测试月份切换。
- [ ] 测试加月份。
- [ ] 测试上传后扫描模拟。
- [ ] 测试识别结果确认入账。
- [ ] 测试不清楚字段留空。
- [ ] 测试新店面确认。
- [ ] 测试疑似重复确认。
- [ ] 测试单元格明细。
- [ ] 测试年度统计。

验收：

- 页面能打开。
- 主要流程能点通。
- 表格合计正确。
- 页面没有明显挤压、重叠、乱码。
- 未验证的能力不说已完成。
