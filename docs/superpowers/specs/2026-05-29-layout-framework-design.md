# 网页端和小程序端整体布局规格

日期：2026-05-29

## 1. 目标

本文件先确定网页端和小程序端的大框架。后续 Claude Code 先按这个框架搭控制面板版面，再逐步实现页面功能。

第一阶段优先做网页端。小程序端先定布局规则，不进入真实开发。

## 2. 总体原则

- 老板一打开就能快速记账。
- 页面入口固定，不让老板找不到功能。
- 表格要清楚，按钮要大，状态要醒目。
- 网页适合看表格和对账，小程序适合快速点选和查看。
- 网页和小程序功能一致，但布局方式不同。

### 2.1 版面密度和按键规则

- 主要信息区要给足空间，尤其是识别结果、价格表、账单明细这类需要核对的数据表格。
- 识别结果表格是核心操作区，版面要比辅助模块更大，文字可以适当缩短，但不能牺牲可读性。
- 非主要信息不要占满整页，比如最近账单、价格匹配提醒、筛选条件、页面步骤等，优先设计成横向按键、标签、折叠区或右侧小面板。
- 后续页面也遵守这个规则：核心表格和核心操作放大，辅助信息收起来或做成按键选项。
- 按键风格参考用户确认过的分段大按钮：当前选中项使用深色底和白字，未选中项使用浅底、细边框、深灰文字。
- 按键颜色可以按业务微调，但不能太花。推荐主色用深墨色或深绿，辅助按钮用浅灰或浅绿，危险操作用红色描边。
- 页面不要堆太多同等重量的卡片。能用按键、标签、顶部筛选条解决的，不要做成大块内容。

## 3. 网页端整体框架

网页端采用后台控制面板布局：

- 左侧固定菜单
- 顶部信息栏
- 右侧主内容区
- 必要时使用弹窗或右侧抽屉处理详情

### 3.1 左侧菜单

左侧菜单固定显示五个入口：

1. 今日记账
2. 价格单
3. 账单汇总
4. 识别记录
5. 设置

默认选中 今日记账。

菜单要求：

- 菜单文字要直白。
- 当前页面要有明显选中状态。
- 菜单宽度固定，不能挤压文字。
- 后续如果有图标，图标只做辅助，不能只显示图标。

### 3.2 顶部信息栏

顶部信息栏显示：

| 区域 | 内容 |
|---|---|
| 左侧 | 当前页面标题，比如今日记账 |
| 中间 | 当前日期，比如2026年5月29日 |
| 右侧 | 门店名称、草稿提示、必要的快捷按钮 |

顶部信息栏建议按钮：

- 今日
- 补记日期
- 草稿箱

顶部信息栏不要放太多操作，避免老板分心。

### 3.3 主内容区

主内容区根据左侧菜单切换。

布局要求：

- 表格页面优先保证横向空间。
- 操作按钮固定在页面上方或表格右侧，不能藏得太深。
- 重要状态用标签显示，比如未付款、部分付款、已付款、未对账、已对账。
- 金额、数量、日期要对齐，方便扫一眼核对。

### 3.4 弹窗和右侧抽屉

网页端建议这样使用：

| 场景 | 展示方式 |
|---|---|
| 新增商铺 | 弹窗 |
| 修改价格 | 弹窗 |
| 批量收款 | 弹窗 |
| 账单详情 | 右侧抽屉 |
| 识别记录详情 | 右侧抽屉 |
| 作废确认 | 确认弹窗 |

原因：

- 弹窗适合短操作。
- 右侧抽屉适合看详情，不打断当前列表。

## 4. 网页端五个页面布局

### 4.1 今日记账

今日记账页面分三层：

1. 商铺选择区
2. 当前记账对象和日期区
3. 三步记账区

#### 商铺选择区

位置：主内容区顶部。

内容：

- 置顶商铺
- 本月销量靠前商铺
- 普通记账
- 添加商铺
- 草稿箱

商铺按钮样式参考用户提供的横向大按钮形式。

按钮排序：

1. 置顶商铺
2. 本月销量从高到低商铺
3. 普通记账
4. 添加商铺
5. 草稿箱

如果商铺太多：

- 第一行显示常用商铺。
- 提供搜索商铺。
- 后续可折叠更多商铺。

#### 当前记账对象和日期区

显示：

| 内容 | 示例 |
|---|---|
| 当前对象 | 王姐超市 |
| 当前日期 | 2026年5月29日 |
| 是否补记 | 正在补记：2026年5月20日 |

补记日期必须明显显示，避免记错日期。

#### 记账流程区

今日记账页面上方先放录入区，录入区下面再放流程步骤。

录入区：

- 点击大的录音按键后，系统自动把识别内容填入核对表格。
- 老板只需要点击大的录音按键，系统自动把识别内容填入核对表格。
- 不需要再单独显示录入步骤，也不需要单独显示开始录入按钮。
- 模拟识别属于第一版内部默认能力或开发测试能力，不作为老板要点击的按钮展示。

页面步骤条只保留：

1. 核对
2. 生成账单

步骤条位置：放在语音录入区下面，识别结果表格标题上方。

录入区内容：

- 左侧或上方是语音入口和文字输入框。
- 语音入口使用大的录音按键，文案为点击录入。
- 页面只保留必要的保存草稿、清空本次录入等辅助操作，不展示开始录入和模拟识别按钮。

核对步骤：

- 以表格为主。
- 每一行显示水果、规格、数量、单位、单价、小计、异常提示。
- 不确定的水果名称显示候选项。
- 核对表格要占据页面主要空间，老板要能一眼看清水果、数量、单价、小计和操作按钮。
- 网页端核对表格要尽量多预留字段列，不能只保留水果、数量、单位、单价、小计这几个基础列。后续输入事项会变多，表格需要容纳规格、价格来源日期、异常提示、备注等信息。
- 如果列较多，优先扩大表格区域或使用横向滚动，不要让辅助模块挤占表格空间。
- 表格底部显示添加一项、共几项、合计金额。合计金额要醒目。
- 最近账单、价格匹配提醒等辅助内容不要抢占核对表格空间，优先放在右侧窄栏、折叠区或按键入口中。

生成账单步骤：

- 显示账单总览。
- 显示付款状态。
- 显示备注。
- 确认生成账单按钮要醒目。
- 如果账单已经确认，提供打印送货单入口。

### 4.2 价格单

价格单页面分三块：

1. 顶部操作区
2. 价格表格
3. 历史价格或别名候选详情

顶部操作区：

- 搜索水果
- 筛选启用/停用
- 新增水果
- 新增规格

价格表格字段：

- 水果名称
- 别名
- 规格
- 当前单价
- 单位
- 生效日期
- 是否启用
- 常用规格
- 操作

右侧抽屉可展示：

- 历史价格
- 别名候选
- 修改记录

### 4.3 账单汇总

账单汇总页面分四块：

1. 筛选区
2. 汇总数字区
3. 视角切换区
4. 账单列表或统计表

筛选区：

- 日期或月份
- 商铺
- 付款状态
- 对账状态

汇总数字区：

- 总金额
- 已收金额
- 待收金额
- 账单数量
- 作废账单数

视角切换：

- 日汇总
- 月汇总
- 按商铺
- 按水果

账单列表操作：

- 查看详情
- 改付款状态
- 批量收款
- 作废
- 导出

账单详情使用右侧抽屉，包含：

- 账单明细
- 调整项
- 付款记录
- 修改记录
- 识别原文
- 打印送货单

### 4.4 识别记录

识别记录页面分三块：

1. 筛选区
2. 记录列表
3. 详情抽屉

筛选区：

- 日期
- 商铺
- 状态

记录列表字段：

- 识别时间
- 商铺
- 记账日期
- 原始文字摘要
- 状态
- 生成账单
- 操作

详情抽屉显示：

- 原始识别文字
- 修正后的表格
- 对应账单
- 操作记录

主要操作：

- 查看
- 继续核对
- 复制成新草稿
- 作废

### 4.5 设置

设置页面使用分组布局。

分组：

1. 门店信息
2. 商铺管理
3. 单位管理
4. 默认规则

设置页面不要做成大段文字说明，要用表单和列表。

商铺管理建议表格：

- 商铺名称
- 联系人
- 是否置顶
- 是否启用
- 本月销售额
- 操作

单位管理：

- 默认单位
- 自定义单位
- 新增单位

默认规则：

- 新账单默认未付款
- 日期默认当天
- 普通记账独立汇总
- 作废代替删除

## 5. 小程序端整体框架

小程序端采用首页入口布局。

首页显示五个大按钮：

1. 今日记账
2. 价格单
3. 账单汇总
4. 识别记录
5. 设置

按钮要大，适合手指点击。

### 5.1 小程序页面顶部

每个独立页面顶部显示：

- 返回按钮
- 页面标题
- 当前日期或筛选日期

页面内不要放太多横向表格，优先使用分块列表和可展开详情。

### 5.2 小程序今日记账

流程：

1. 进入今日记账
2. 选择商铺或普通记账
3. 选择或确认日期
4. 录入
5. 核对
6. 生成账单

商铺展示：

- 置顶商铺在最上面
- 本月销量高的商铺靠前
- 普通记账固定显示
- 添加商铺固定显示

核对表格在小程序端改成一行一个水果卡片：

- 水果
- 规格
- 数量
- 单位
- 单价
- 小计
- 异常提示

### 5.3 小程序价格单

价格单采用列表：

- 水果名称
- 当前常用规格
- 当前价格
- 单位

点开水果后看：

- 所有规格
- 历史价格
- 别名候选

### 5.4 小程序账单汇总

账单汇总采用分段切换：

- 日汇总
- 月汇总
- 商铺
- 水果

列表优先显示：

- 商铺
- 日期
- 金额
- 付款状态

点开后看账单详情。

### 5.5 小程序识别记录

识别记录采用列表：

- 时间
- 商铺
- 原始文字摘要
- 状态

点开查看详情，可继续核对或复制成新草稿。

### 5.6 小程序设置

小程序设置用分组列表：

- 门店信息
- 商铺管理
- 单位管理
- 默认规则

每个分组点开后进入独立设置页面。

## 6. 第一阶段实现建议

第一阶段只实现网页端控制面板。

推荐顺序：

1. 搭网页端外框：左侧菜单、顶部信息栏、主内容区。
2. 做今日记账页面布局。
3. 做价格单页面布局。
4. 做账单汇总页面布局。
5. 做识别记录页面布局。
6. 做设置页面布局。
7. 加入前端假数据和基础交互。
8. 用浏览器检查页面打开、菜单切换、按钮点击和移动端宽度。

小程序端只保留布局规格，等网页端稳定后再实现。

## 7. 交给 Claude Code 的布局实现边界

Claude Code 第一轮只做版面和基础交互，不接后端。

必须交付：

- 网页端左侧菜单
- 顶部信息栏
- 五个页面的可切换内容
- 今日记账商铺按钮区和三步流程
- 价格单表格
- 账单汇总筛选、汇总数字、视角切换和列表
- 识别记录列表和详情区域
- 设置分组页面

暂不要求：

- 真实语音识别
- 真实数据库
- 真实导出文件
- 真实打印机硬件连接
- 真实小程序
- 自动发送消息
- 付款扣费

## 8. 验证标准

Claude Code 完成后，必须验证：

1. 网页能打开。
2. 默认进入今日记账。
3. 左侧五个菜单都能切换。
4. 今日记账商铺按钮能选中。
5. 三步流程能切换。
6. 价格单表格能显示多规格。
7. 账单汇总能切换日汇总、月汇总、按商铺、按水果。
8. 识别记录能打开详情。
9. 设置页分组清楚。
10. 手机宽度下页面不重叠、不挤出屏幕。
