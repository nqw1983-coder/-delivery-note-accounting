## 项目方向

项目核心是扫描手写送货单记账，不做语音记账。

第一阶段先做网页版控制面板，Mac 电脑和苹果手机浏览器都能使用。功能稳定后，再考虑微信小程序、苹果端、安卓端。

## 已确认的核心规则

1. 每个月一个表格页面。
2. 表格左侧是日期，顶部是店面名称，中间格子是金额。
3. 每个格子默认显示 0。
4. 扫描手写送货单后，识别店面、日期、总金额。
5. 识别清楚时自动填入对应月份、对应日期、对应店面。
6. 识别不清楚时必须留空，不能猜，不能瞎填，不能产生虚假信息。
7. 空白信息由用户人工填写或修改。
8. 同一天同店面多张单，格子显示总数。
9. 点开格子只看加法明细，比如 52 + 165 = 217。
10. 第一版不保存原图。
11. 第一版数据结构不保存完整送货单，只保存月表格金额和必要的加法明细。
12. 修改金额后只保留最新金额。
13. 同一天、同店面、同金额时提示可能重复，用户确认后才加入。

## 控制面板布局

左侧：

1. 标题：送货单记账。
2. 标题旁放年度统计按钮。
3. 标题旁放 + 按钮，用来增加当前年份的下一个月份。
4. 年份选择先固定为 2026、2027、2028。
5. 年份选择旁放搜索月份。
6. 左侧只放月份列表，不放待处理入口。

右侧：

1. 显示当前月份标题，比如 2026年5月。
2. 月份标题下面放上传文件框和开始扫描按钮。
3. 主区域显示当前月份表格。
4. 不要笔记编辑工具栏，不要颜色按钮，不要复制文本、导出、打印按钮。

年度统计：

1. 点击年度统计按钮打开弹窗。
2. 弹窗显示当前年份每个月总金额。
3. 每个月总金额自动从月表合计汇总。
4. 第一版先统计当前选择年份。

月份新增规则：

1. 点击 + 自动新增当前年份的下一个月份。
2. 如果当前年份没有月份，先新增 1 月。
3. 如果已经有 12 月，再点 + 时提示当前年份已满 12 个月。
4. 新月份排到月份列表最上面。

店面规则：

1. 第一版默认店面：吾湘、吾黄、醉得意、萍姐、柳。
2. 新月份自动带出默认店面列。
3. 扫描到已有店面，自动填入对应列。
4. 扫描到新店面，先提示确认，确认后新增列。
5. 不需要单独做店面管理页面。
