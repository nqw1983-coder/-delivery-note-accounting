# 水果店手写送货单记账

一个面向水果店、餐饮供货、线下小店的手写送货单记账网页工具。

老板上传或扫描手写送货单图片后，系统识别店面、日期和总金额，并把金额填入对应月份、对应日期、对应店面的表格里。识别不清楚的信息会留空，提示人工补充，避免把不确定内容写成假账。

## 项目目标

- 用浏览器完成手写送货单记账，先支持 Mac 电脑和苹果手机浏览器。
- 第一版重点识别店面、日期、总金额，不强制识别水果明细。
- 识别清楚才自动填表，识别不清楚必须人工确认。
- 同一天、同店面、同金额时提示可能重复，确认后再入账。
- 表格要简单，适合不熟悉复杂软件的店主日常使用。

## 当前功能

- 月份列表和月份金额表格。
- 默认店面列：吾湘、吾黄、醉得意、萍姐、柳。
- 上传多张送货单图片。
- OCR 识别店面、日期、总金额和备注。
- 识别结果确认弹窗，可人工修改店面、日期、金额、备注。
- 同日同店同金额的重复提醒。
- 新店面确认后加入表格。
- 点击金额格子查看加法明细。
- 年度统计弹窗。
- 手机浏览器基础适配。

## 技术栈

- React
- TypeScript
- Vite
- lucide-react

OCR 目前支持通义千问 VL、豆包 Vision 的配置入口。真实使用时需要用户自己配置对应服务的 API Key。

## 本地运行

先安装依赖：

```bash
npm install
```

启动开发页面：

```bash
npm run dev
```

打包检查：

```bash
npm run build
```

## 使用方式

1. 打开网页后选择当前月份。
2. 上传一张或多张手写送货单图片。
3. 点击开始扫描。
4. 在识别确认弹窗里检查店面、日期、金额。
5. 信息正确后点击确认入账。
6. 如果识别不清楚，手动补全后再入账。

## 数据说明

当前版本主要是前端网页原型，数据保存在浏览器本地状态或本地存储中，还没有正式后端。请不要把它当成已经可长期保存账本的正式系统。

后续计划增加：

- 更可靠的数据保存。
- 原图追溯。
- 修改记录。
- 待处理单据。
- 更完整的移动端体验。
- 更系统的测试。

## 安全提醒

送货单图片可能包含客户名、店铺名、金额等信息。接入外部 OCR 服务前，请确认自己接受对应服务的数据处理方式，不要上传身份证、银行卡、密码、密钥等敏感信息。
